## 应用初始化流程

```
main.tsx
  └─ App.tsx
       ├─ GlassParamsProvider（玻璃效果参数 Context）
       └─ AppInner
            ├─ initAudio()           → 初始化 HTMLAudioElement
            ├─ loadLocalSongs()      → 从 /musics/index.json 加载本地歌曲
            ├─ loadUserPlaylists()   → 登录后从 localStorage 加载用户歌单
            └─ 渲染页面组件
                 ├─ AppleMusicDiscover（主页面）
                 ├─ AlbumPage（条件渲染）
                 ├─ LyricsPage（条件渲染）
                 └─ LoginModal（条件渲染）
```

## 页面切换机制

项目**未使用 react-router**，所有页面切换通过 `useState` 控制：

- `showLyrics` → 显示/隐藏 [[音频与歌词|歌词页]]
- `showAlbum` + `albumData` → 显示/隐藏 [[页面与组件|专辑页]]，并传递专辑数据
- `showLoginModal` → 显示/隐藏登录弹窗（由 `useAuthStore` 控制）

## 构建配置要点

- **Base 路径**：`/wanshui-music/`（GitHub Pages 部署需要）
- **路径别名**：`@/` → `src/`（通过 `vite-tsconfig-paths` 插件实现）
- **Source Map**：构建时生成 `hidden` 类型 sourcemap
- **Babel 插件**：`react-dev-locator`（开发时元素定位）
- **Trae Badge**：生产构建时右下角显示 Trae 标识

## 静态资源约定

> 详见 [[状态管理]] 中的数据加载部分

- 远程歌曲：封面和音频使用外部 URL（Apple Music CDN + SoundHelix 示例音频）
- 本地歌曲：存放在 `public/musics/` 目录，通过 `musics/index.json` 索引
- **所有本地资源路径必须使用 `import.meta.env.BASE_URL` 前缀**，否则部署后会出现 404
