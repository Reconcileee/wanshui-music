# 状态管理

> 相关文档：[[架构概览]] | [[页面与组件]] | [[音频与歌词]]

项目使用 **Zustand 5** 进行状态管理，共有三个 Store。其中两个是标准 Zustand Store，一个是基于 React Context 的参数管理。

## Store 总览

| Store | 文件 | 类型 | 职责 |
|-------|------|------|------|
| `useMusicStore` | `store/useMusicStore.ts` | Zustand | 音乐播放、歌曲列表、用户歌单 |
| `useAuthStore` | `store/useAuthStore.ts` | Zustand | 用户认证、登录状态 |
| `useGlassParams` | `store/useGlassParams.tsx` | React Context | 液态玻璃效果参数 |

---

## useMusicStore

核心状态 Store，管理播放器所有运行时状态。

### 状态字段

```
MusicState
├── 播放状态
│   ├── isPlaying: boolean        # 是否正在播放
│   ├── currentSong: Song | null  # 当前播放歌曲
│   ├── currentIndex: number      # 当前歌曲在播放列表中的索引
│   ├── progress: number          # 播放进度 (0-1)
│   └── activeTab: TabType        # 当前激活的导航标签
├── 播放列表
│   ├── playlist: Song[]          # 当前播放队列
│   └── audioRef: HTMLAudioElement | null  # 音频实例引用
├── 加载状态
│   └── loadedAudioUrl: string | null  # 已加载的音频 URL（避免重复加载）
└── 用户歌单
    ├── userPlaylists: UserPlaylist[]  # 用户自定义歌单列表
    └── currentUsername: string | null # 当前登录用户名
```

### 核心方法

#### 播放控制

| 方法 | 说明 |
|------|------|
| `initAudio()` | 创建 `HTMLAudioElement` 实例，绑定 `ended` 和 `timeupdate` 事件 |
| `play()` | 播放当前歌曲，自动处理音频源切换 |
| `pause()` | 暂停播放 |
| `toggle()` | 切换播放/暂停 |
| `next()` | 播放下一首（循环） |
| `prev()` | 播放上一首（循环） |
| `seek(progress)` | 跳转到指定进度 (0-1) |

#### 歌曲播放

| 方法 | 说明 |
|------|------|
| `playSong(index)` | 按索引播放播放列表中的歌曲 |
| `playSongObject(song)` | 按 Song 对象播放；若歌曲不在列表中则追加到末尾 |

> **重要**：`playSongObject` 是推荐的歌曲播放方式，因为它能处理歌曲不在当前播放列表中的情况。`playSong` 的 `findIndex` 可能因歌曲不在列表而失败。

#### 歌单管理

| 方法 | 说明 |
|------|------|
| `loadUserPlaylists(username)` | 从 localStorage 加载用户歌单 |
| `createPlaylist(name)` | 创建新歌单 |
| `deletePlaylist(playlistId)` | 删除歌单（不可删除默认的"喜欢的歌曲"） |
| `renamePlaylist(playlistId, newName)` | 重命名歌单 |
| `addToPlaylist(playlistId, song)` | 添加歌曲到歌单（自动去重） |
| `removeFromPlaylist(playlistId, songId)` | 从歌单移除歌曲 |
| `clearUserPlaylists()` | 清空歌单状态（登出时调用） |

### 数据流向

```
App.tsx (初始化)
  │
  ├─ initAudio() ──→ 创建 Audio 实例
  │                    ├─ on ended → next()
  │                    └─ on timeupdate → progress = currentTime / duration
  │
  ├─ loadLocalSongs() ──→ fetch /musics/index.json
  │                       └─ addSongsToPlaylist() ──→ 追加到 playlist（去重）
  │
  └─ 登录状态变化
       ├─ 登录 → loadUserPlaylists(username) ──→ 从 localStorage 读取
       └─ 登出 → clearUserPlaylists()
```

### 持久化策略

用户歌单通过 **localStorage** 持久化：

- Key 格式：`music-player-playlist-{username}`
- 每次歌单变更（创建/删除/重命名/添加/移除）都会同步写入 localStorage
- 默认歌单：每个用户自动拥有"喜欢的歌曲"歌单（id: `favorites`），不可删除

## useAuthStore

管理用户认证状态。

### 状态字段

```
AuthState
├── isLoggedIn: boolean      # 是否已登录
├── showLoginModal: boolean  # 是否显示登录弹窗
├── username: string         # 当前用户名
├── isAdmin: boolean         # 是否管理员
└── loginError: string       # 登录错误信息
```

### 认证逻辑

```
login(username, password)
  │
  ├─ 匹配管理员账户 (Fennmoo / ////////5056)
  │   └─ isAdmin = true
  │
  ├─ 普通用户（仅验证用户名非空）
  │   └─ isAdmin = false
  │
  └─ 用户名为空
      └─ 返回 false，设置 loginError
```

### 持久化

- Key：`music-player-auth`
- 保存字段：`isLoggedIn`、`username`、`isAdmin`
- 登录/登出时自动同步到 localStorage
- 应用启动时从 localStorage 恢复状态

### 与其他 Store 的联动

在 `App.tsx` 中，`useAuthStore` 的登录状态变化会触发 `useMusicStore` 的操作：

```typescript
// 登录 → 加载用户歌单
if (isLoggedIn && username) {
  loadUserPlaylists(username);
}
// 登出 → 清除歌单
else {
  clearUserPlaylists();
}
```

## useGlassParams (Context)

基于 React Context 的参数管理，用于控制 [[液态玻璃效果]] 的视觉参数。

### 为什么用 Context 而非 Zustand？

玻璃参数需要被 `LiquidGlassControls` 面板修改，同时被 `MiniPlayer` 等多个组件消费。使用 Context 更符合 React 组件树的天然层级关系，且参数变更频率低，不需要 Zustand 的细粒度订阅优化。

### 数据结构

```typescript
GlassParams {
  // 形状
  radius: 60              // 圆角半径
  superellipseN: 5        // 超椭圆指数
  shape: 'superellipse'   // 形状类型

  // 折射
  glassThickness: 80      // 玻璃厚度
  bezelWidth: 60          // 边框宽度
  ior: 3.0                // 折射率
  scaleRatio: 1.0         // 缩放比

  // 外观
  blurAmount: 0.3         // 模糊量
  specularOpacity: 0.5    // 高光透明度
  specularSaturation: 4   // 高光饱和度
  effect: 'regular'       // 效果模式 (regular | clear)

  // 阴影
  shadowBlur: 20          // 内阴影模糊
  shadowSpread: -5        // 内阴影扩散
  outerShadowBlur: 24     // 外阴影模糊

  // 着色
  tintOpacity: 0.06       // 着色透明度
  tintR/G/B: 255          // 着色 RGB
}
```

### 使用方式

```typescript
const { params, setParams } = useGlassParams();
```

## 数据模型

### Song

```typescript
interface Song {
  id: number;
  title: string;
  artist: string;
  album: string;
  cover: string;          // 封面图 URL
  audioUrl: string;       // 音频 URL
  duration: number;       // 时长（秒）
  lyricsUrl?: string;     // LRC 歌词文件 URL
  lyricsSrtUrl?: string;  // SRT 字幕文件 URL
}
```

### UserPlaylist

```typescript
interface UserPlaylist {
  id: string;             // 歌单 ID（favorites 为默认）
  name: string;           // 歌单名称
  songs: Song[];          // 歌曲列表
  createdAt: number;      // 创建时间戳
}
```

### 数据加载

歌曲数据来自两个来源：

1. **远程歌曲**：硬编码在 `mockData.ts` 的 `remoteSongs` 数组中
2. **本地歌曲**：通过 `loadLocalSongs()` 异步加载 `public/musics/index.json`

本地歌曲的路径处理规则：
- 如果 URL 以 `http` 开头 → 保持原样
- 否则 → 添加 `import.meta.env.BASE_URL` 前缀

> 这是因为 Vite 的 `base` 配置为 `/wanshui-music/`，不添加前缀会导致部署后资源 404。
