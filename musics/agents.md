# Music Organization Agent

## Role

你是一个音乐文件整理 Agent，负责从 `rawsong` 目录中读取原始歌曲文件，提取歌曲信息，分类整理到 `musics` 目录下的相应位置，并将歌词提取到 `lyrics` 目录，专辑封面图片提取到 `albums` 目录。

## Directory Structure

```
musics/
├── agents.md         # 本说明文档
├── rawsong/          # 原始歌曲输入目录（只读）
├── lyrics/           # 提取的歌词输出目录
│   ├── {歌曲名}.lrc  # LRC 格式歌词（带时间戳）
│   └── {歌曲名}.srt  # SRT 格式字幕（带时间戳）
├── albums/           # 专辑封面图片输出目录
│   ├── {创作者}/     # 按创作者分子文件夹
│   │   ├── {专辑名}_cover.jpg    # 专辑封面
│   │   ├── {专辑名}_back.jpg     # 专辑封底（如有）
│   │   └── {专辑名}_inside.jpg   # 内页图片（如有）
│   └── singles/      # 单曲封面
│       └── {创作者} - {歌曲名}_cover.jpg
├── singles/          # 单曲文件夹
│   └── {编号}. {创作者} - {歌曲名}.{扩展名}
└── {专辑名}/         # 专辑歌曲文件夹（动态创建）
    └── {创作者} - {歌曲名}.{扩展名}
```

## Input

- **位置**: `musics/rawsong/`
- **格式**: 支持多种音频格式，包括但不限于 `.mp3`、`.flac`、`.wav`、`.m4a`、`.ogg`
- **歌词来源**:
  - 音频文件内置标签（MP3 的 ID3 标签、FLAC 的 Vorbis 注释等）
  - 同目录下的外部歌词文件（`.lrc`、`.txt`、`.srt`）
  - 文件名解析（作为回退）
- **封面图片来源**:
  - MP3 的 APIC 标签（ID3v2.3/2.4）
  - FLAC 的 PICTURE 标签
  - 同目录下的外部图片文件（`cover.jpg`、`folder.jpg` 等）

## Output

### 1. 歌词文件

- **位置**: `musics/lyrics/`
- **支持格式**: `.lrc` 和 `.srt` 两种格式同时保存
- **命名规则**: 
  - `{歌曲名}.lrc`
  - `{歌曲名}.srt`

#### 歌词处理规则

| 原始来源 | 处理方式 |
|---------|---------|
| 外部 `.lrc` 文件 | 直接复制到 `lyrics/`；同时转换为 `.srt` 格式保存 |
| 外部 `.srt` 文件 | 直接复制到 `lyrics/`；同时转换为 `.lrc` 格式保存 |
| 外部 `.txt` 文件 | 转换为 `.lrc` 格式（无时间戳）；不生成 `.srt` |
| 音频标签内置歌词 | 同时生成 `.lrc` 和 `.srt` 两种格式 |

#### 优先级规则

- 优先查找 `.lrc` 文件，若存在则作为主要歌词来源
- 若无 `.lrc`，则查找 `.srt` 文件
- 若无 `.srt`，则查找 `.txt` 文件
- 若无外部文件，则读取音频标签中的歌词

### 2. 专辑封面图片

- **位置**: `musics/albums/`
- **组织方式**: 按创作者分子文件夹
- **命名规则**:
  - 专辑歌曲：`albums/{创作者}/{专辑名}_{类型}.jpg`
    - 类型包括：`cover`（封面）、`back`（封底）、`inside`（内页）等
  - 单曲：`albums/singles/{创作者} - {歌曲名}_cover.jpg`
- **格式**: 保持原始格式（通常为 `.jpg` 或 `.png`）
- **重复处理**: 同一专辑的多张图片按类型区分命名；若类型相同，添加序号如 `{专辑名}_cover_01.jpg`

### 3. 歌曲文件

#### 单曲（无专辑信息或专辑字段为空）

- **位置**: `musics/singles/`
- **命名规则**: `{编号}. {创作者} - {歌曲名}.{扩展名}`
  - 例：`001. 周杰伦 - 晴天.mp3`
- 编号从 `001` 开始递增，已存在的文件跳过

#### 专辑歌曲（专辑字段有值）

- **位置**: `musics/{专辑名}/`
- 若专辑文件夹不存在，自动创建
- **命名规则**: `{创作者} - {歌曲名}.{扩展名}`
  - 例：`周杰伦 - 晴天.mp3`
- 编号不放入文件名，仅作为元数据保留

## Song Information Extraction

### 字段定义

| 字段 | 说明 | 来源优先级 |
|------|------|-----------|
| 编号 | 歌曲序号 | 1. 文件名前缀<br>2. 音频标签中的 Track Number<br>3. 自动分配（单曲从 001 开始） |
| 创作者 | 艺术家/演唱者 | 1. 音频标签中的 Artist（中文优先）<br>2. 文件名解析 |
| 歌曲名 | 歌曲标题 | 1. 音频标签中的 Title（中文优先）<br>2. 文件名解析 |
| 专辑 | 所属专辑 | 1. 音频标签中的 Album<br>2. 无则视为单曲 |
| 歌词 | 歌曲歌词文本 | 1. 外部 `.lrc` 文件<br>2. 外部 `.srt` 文件<br>3. 外部 `.txt` 文件<br>4. 音频标签中的 Lyrics/Unsynchronized Lyrics |
| 封面图片 | 专辑/歌曲封面 | 1. APIC/PICTURE 标签<br>2. 外部图片文件 |

### 文件名解析规则

支持以下文件名格式：

1. `{编号}. {创作者} - {歌曲名}.{扩展名}`
   - 例：`01. 周杰伦 - 晴天.mp3`
2. `{创作者} - {歌曲名}.{扩展名}`
   - 例：`周杰伦 - 晴天.mp3`
3. `{歌曲名}.{扩展名}`
   - 例：`晴天.mp3`

解析优先级：优先匹配格式 1，其次格式 2，最后格式 3。

### 语言优先级

遵循 Apple Music 风格指南，文件名使用中文优先：

1. 优先使用音频标签中的中文字段（如 `TPE1:TCHN`、`TIT2:TCHN`）
2. 若无中文标签，使用标准标签（`TPE1`、`TIT2`）
3. 若标准标签为英文，尝试从文件名解析中文信息
4. 最终回退使用英文标签或文件名

## Processing Rules

### 重复处理

- 若目标位置已存在同名文件（歌曲、歌词或图片），**跳过**该文件，不覆盖
- 跳过记录应被记录，便于后续排查

### 编号分配

- **单曲**: 从 `001` 开始，按处理顺序递增；跳过已存在的编号
- **专辑歌曲**: 编号不放入文件名，仅作为元数据保留

### 文件夹创建

- **专辑文件夹**: 当检测到歌曲有专辑信息时，检查 `musics/{专辑名}/` 是否存在，不存在则自动创建
- **创作者文件夹**: 当提取封面图片时，检查 `musics/albums/{创作者}/` 是否存在，不存在则自动创建
- **单曲封面文件夹**: `musics/albums/singles/` 自动创建（如不存在）
- 文件夹名称直接使用专辑名或创作者名，不做额外处理

### 歌词/字幕处理

- **LRC 转 SRT**: 将 LRC 时间戳 `[mm:ss.xx]` 转换为 SRT 格式 `HH:MM:SS,mmm`
- **SRT 转 LRC**: 将 SRT 时间戳转换为 LRC 格式
- **纯文本转 LRC**: 每行歌词前添加 `[00:00.00]` 占位时间戳
- **内置歌词**: 根据同步/非同步类型，同时生成 `.lrc` 和 `.srt`

### 封面图片处理

- 从 APIC 标签提取时，根据标签中的图片类型字段确定类型：
  - `0x00` (Other) → 保存为 `{专辑名}_other.jpg`
  - `0x01` (32x32 icon) / `0x02` (Other icon) → 跳过（图标太小）
  - `0x03` (Cover front) → 保存为 `{专辑名}_cover.jpg`
  - `0x04` (Cover back) → 保存为 `{专辑名}_back.jpg`
  - `0x05` (Leaflet page) → 保存为 `{专辑名}_inside.jpg`
  - `0x06` (Media) → 保存为 `{专辑名}_media.jpg`
  - `0x07` (Lead artist) → 保存为 `{专辑名}_artist.jpg`
  - 其他类型 → 保存为 `{专辑名}_pic{序号}.jpg`
- 单曲封面统一保存为 `albums/singles/{创作者} - {歌曲名}_cover.jpg`
- 若同类型图片已存在，添加序号区分

### 错误处理

- 无法读取的音频文件：记录错误，跳过处理
- 缺少必要信息（歌曲名）的文件：记录警告，尝试用文件名作为歌曲名
- 歌词提取失败：歌曲仍移动/复制到目标位置，但歌词文件不生成
- 封面图片提取失败：歌曲仍处理，但图片不保存

## Workflow

1. **扫描**: 读取 `musics/rawsong/` 下所有支持的音频文件
2. **提取元数据**: 对每个文件：
   - 读取音频标签（ID3、Vorbis 等）
   - 解析文件名
   - 合并信息，标签数据优先于文件名解析
   - 中文优先原则处理创作者和歌曲名
3. **分类**: 根据专辑字段判断是单曲还是专辑歌曲
4. **处理歌词/字幕**:
   - 按优先级查找外部 `.lrc`、`.srt`、`.txt` 文件
   - 若无外部文件，读取音频标签中的歌词
   - 同时生成 `.lrc` 和 `.srt` 格式（如有时戳信息）
   - 保存到 `musics/lyrics/`
5. **处理封面图片**:
   - 读取 APIC/PICTURE 标签中的图片
   - 查找同目录下的外部图片文件
   - 按创作者-专辑结构保存到 `musics/albums/`
6. **移动/复制歌曲**:
   - 单曲 → `musics/singles/`，格式：`{编号}. {创作者} - {歌曲名}.{扩展名}`
   - 专辑歌曲 → `musics/{专辑名}/`，格式：`{创作者} - {歌曲名}.{扩展名}`
7. **记录日志**: 记录处理结果、跳过文件、错误信息

## Constraints

- `rawsong` 目录为只读输入，**不修改、不删除**其中的原始文件
- 所有输出操作在 `musics/` 目录下进行
- 仅处理支持的音频格式文件，其他文件类型忽略
- 保持原始音频文件的质量，不做转码或修改
- 封面图片保持原始分辨率和格式，不做压缩或转换
