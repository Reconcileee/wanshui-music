## 页面层级

项目不使用 react-router，页面切换通过 `App.tsx` 中的 `useState` 控制，所有页面以**条件渲染**方式呈现。

```
App.tsx
├── AppleMusicDiscover    ← 始终显示（主页面）
├── AlbumPage             ← showAlbum 为 true 时覆盖显示
├── LyricsPage            ← showLyrics 为 true 时覆盖显示
└── LoginModal            ← showLoginModal 为 true 时弹窗显示
```

### 页面切换关系

```
AppleMusicDiscover
  │
  ├─ 点击 FeaturedCard 播放按钮 ──→ onPlay() + onClick()
  │   └─ onClick() ──→ onOpenAlbum() ──→ 显示 AlbumPage
  │
  ├─ 点击歌曲列表项 ──→ 播放歌曲
  │
  └─ 点击歌词入口 ──→ onOpenLyrics() ──→ 显示 LyricsPage

AlbumPage
  └─ 点击返回 ──→ onClose() ──→ 回到 AppleMusicDiscover

LyricsPage
  └─ 点击收起 ──→ onClose() ──→ 回到 AppleMusicDiscover
```

## 页面详解

### AppleMusicDiscover

**文件**：`pages/AppleMusicDiscover.tsx`

核心发现页面，是应用的主界面。包含以下区域：

| 区域 | 说明 |
|------|------|
| Sidebar | 左侧固定导航栏（256px 宽） |
| 主内容区 | 新歌速递、推荐卡片、歌曲列表等 |
| MiniPlayer | 可拖拽的迷你播放器（浮层） |
| BottomNav | 移动端底部导航 |
| ProgressBar | 带悬浮预览的进度条 |

**关键组件**：

- `FeaturedCard` — 大尺寸推荐卡片，点击播放按钮同时触发 `onPlay` 和 `onClick`（导航到专辑页）
- `NEW_SONGS` — 新歌速递数组，新增歌曲需同时更新此数组和 `mockData.ts` 中的 `mockSongs`
- `ProgressBar` — 悬浮提示框包含封面缩略图、歌曲信息、时间差等

### AlbumPage

**文件**：`pages/AlbumPage.tsx`

专辑详情页，完全复刻 Apple Music 专辑页风格。

**数据结构**：

```typescript
AlbumData {
  title, artist, genre, year, releaseDate
  trackCount, totalDuration, label
  cover, description
  tracks: AlbumTrack[]           // 曲目列表
  videos?: Video[]               // 音乐视频
  otherVersions?: Card[]         // 其他版本
  moreFromArtist?: Card[]        // 更多艺人作品
  appearsIn?: Card[]             // 出现在以下内容
  youMayAlsoLike?: Card[]        // 你可能也喜欢
}
```

**内置专辑数据**：

- `OLIVIA_ALBUM` — Olivia Rodrigo 的 "you seem pretty sad for a girl so in love"
- `SHOUDU_ALBUM` — 罗大佑的 "首都"

**页面布局**：

1. 返回按钮（sticky 顶部）
2. 专辑头部：封面（270x270，无圆角）+ 元信息 + 描述 + 操作按钮
3. 曲目列表：序号/热度点、标题、时长、试听按钮
4. 版权信息
5. 其他版本 / 音乐视频 / 更多艺人作品 / 出现在 / 你可能也喜欢

**播放逻辑**：

- 点击曲目 → 构造 `Song` 对象（id = track.id + 5000 避免冲突）→ 调用 `playSongObject`
- 音频 URL 使用 SoundHelix 示例音频占位

### LyricsPage

**文件**：`pages/LyricsPage.tsx`

全屏歌词页，Apple Music 风格。

**布局**：

- 左侧：专辑封面 + 歌曲信息 + 进度条 + 播放控制 + 音量
- 右侧：滚动歌词（仅桌面端显示）

**歌词交互**：

- 当前行高亮（30px 白色粗体）
- 已唱行淡出（25% 透明度）
- 未唱行半透明（40% 透明度）
- 自动滚动到当前行
- 歌词加载详见 [[音频与歌词]]

## 组件详解

### Sidebar

**文件**：`components/Sidebar.tsx`

固定定位的左侧导航栏，Apple Music 风格毛玻璃效果。

**结构**：

```
Sidebar (fixed, 232px, 圆角 20px)
├── Logo（Apple Music 图标 + "Music" 文字）
├── 搜索按钮
├── 主导航（主页、新发现、广播）
├── 资料库（最近添加、艺人、专辑、歌曲）
├── 歌单区
│   ├── 新建歌单（登录后可见）
│   └── 用户歌单列表（可重命名/删除）
└── 底部
    ├── "在音乐中打开" 链接
    └── 登录/退出按钮
```

**歌单管理**：

- 新建歌单：输入框 + 确认/取消按钮
- 重命名：内联编辑模式
- 删除：hover 显示删除按钮，默认"喜欢的歌曲"不可删除
- 未登录时显示"登录后可管理歌单"

### MiniPlayer

**文件**：`components/MiniPlayer.tsx`

可拖拽的迷你播放器，使用 [[液态玻璃效果]] 包裹。

**特性**：

- 通过 `useDraggable` Hook 实现拖拽，支持惯性和边界碰撞反弹
- 播放时封面旋转动画（`requestAnimationFrame` 驱动）
- 显示歌曲名、艺人名、进度条
- 播放/暂停 + 下一首按钮

### LoginModal

**文件**：`components/LoginModal.tsx`

登录弹窗，受 `useAuthStore.showLoginModal` 控制。

## 自定义 Hooks

### useDraggable

**文件**：`hooks/useDraggable.ts`

为元素添加拖拽能力，返回 `ref` 绑定到目标元素。

**特性**：

- Pointer Events 实现，支持鼠标和触摸
- 边界约束（`boundaryPadding`）
- 惯性滑动（摩擦系数 0.92）
- 边界碰撞反弹（速度反转 × 0.3）

### useLiquidGlass

**文件**：`hooks/useLiquidGlass.ts`

为元素绑定 SVG 液态玻璃滤镜，详见 [[液态玻璃效果]]。

### useTheme

**文件**：`hooks/useTheme.ts`

主题管理 Hook。
