## 播放引擎

项目使用浏览器原生 **HTMLAudioElement** 作为播放引擎，不依赖第三方音频库。

### 初始化

在 `App.tsx` 中通过 `useMusicStore.initAudio()` 创建全局唯一的 `Audio` 实例：

```typescript
initAudio: () => {
  const audio = new Audio();
  // 播放结束 → 自动下一首
  audio.addEventListener('ended', () => get().next());
  // 时间更新 → 同步进度
  audio.addEventListener('timeupdate', () => {
    if (audio.duration) {
      set({ progress: audio.currentTime / audio.duration });
    }
  });
  set({ audioRef: audio });
}
```

### 播放控制流程

```
用户操作
  │
  ├─ playSongObject(song)
  │     ├─ 歌曲在播放列表中？ → playSong(index)
  │     └─ 不在列表中？ → 追加到末尾 → playSong(newIndex)
  │
  ├─ playSong(index)
  │     └─ 设置 currentIndex/currentSong → audio.src = song.audioUrl → audio.play()
  │
  ├─ toggle()
  │     ├─ isPlaying → pause()
  │     └─ !isPlaying → play()
  │
  ├─ seek(progress)
  │     └─ audio.currentTime = progress * duration
  │
  ├─ next()
  │     └─ currentIndex = (currentIndex + 1) % playlist.length
  │
  └─ prev()
        └─ currentIndex = (currentIndex - 1 + length) % length
```

### 音频源管理

`loadedAudioUrl` 字段跟踪当前已加载的音频 URL，避免重复设置 `audio.src`：

```typescript
play: () => {
  if (currentSong && loadedAudioUrl !== currentSong.audioUrl) {
    audio.src = currentSong.audioUrl;  // 仅在 URL 变化时更新
    set({ loadedAudioUrl: currentSong.audioUrl });
  }
  audio.play();
}
```

### 进度同步

播放进度通过 `timeupdate` 事件实时同步到 Store：

- `progress`：0-1 的归一化值（`currentTime / duration`）
- 组件通过 `useMusicStore(s => s.progress)` 订阅进度更新
- `seek()` 方法接受 0-1 的归一化值，内部转换为实际时间

### 音频来源

| 来源 | URL 格式 | 说明 |
|------|---------|------|
| 远程示例 | `https://www.soundhelix.com/examples/mp3/SoundHelix-Song-N.mp3` | 占位音频 |
| 本地文件 | `{BASE_URL}musics/xxx.mp3` | 需要添加 `import.meta.env.BASE_URL` 前缀 |

> **注意**：`crossOrigin` 属性仅用于外部 HTTP URL，本地文件不应设置此属性，否则会导致 CORS 问题。

## 歌词系统

### 歌词数据来源

歌曲的 `Song` 类型包含两个可选的歌词 URL 字段：

```typescript
interface Song {
  // ...
  lyricsUrl?: string;      // LRC 格式歌词
  lyricsSrtUrl?: string;   // SRT 格式字幕
}
```

加载优先级：`lyricsUrl` > `lyricsSrtUrl`，即优先使用 LRC 格式。

### 歌词加载流程

在 [[页面与组件|LyricsPage]] 中，当 `currentSong` 变化时触发歌词加载：

```
currentSong 变化
  │
  ├─ 无歌词 URL？ → 显示"暂无歌词"
  │
  └─ 有歌词 URL
       ├─ fetch(url)
       ├─ 解析文本
       │    ├─ LRC → parseLRC()
       │    └─ SRT → parseSRT()
       ├─ 解析成功 → 更新 lyrics 状态
       └─ 解析失败 → 显示"暂无歌词"
```

### LRC 格式解析

`parseLRC()` 解析标准 LRC 歌词格式：

```
[00:12.50]第一句歌词
[00:15.30]第二句歌词
[01:02.100]第三句歌词
```

**解析规则**：

- 匹配 `[mm:ss.xx]` 时间标签，支持 1-3 位小数
- 移除所有时间标签后提取歌词文本
- 按时间升序排序

**输出**：

```typescript
interface LyricLine {
  time: number;   // 秒（含小数）
  text: string;   // 歌词文本
}
```

### SRT 格式解析

`parseSRT()` 解析 SubRip 字幕格式：

```
1
00:00:22,670 --> 00:00:30,530
歌词文本
```

**解析规则**：

- 按空行分割块
- 提取起始时间（忽略结束时间）
- 支持多行歌词文本（用空格连接）

### 歌词同步

`findCurrentLineIndex()` 根据当前播放时间定位歌词行：

```typescript
function findCurrentLineIndex(lyrics: LyricLine[], currentTime: number): number {
  let index = 0;
  for (let i = 0; i < lyrics.length; i++) {
    if (currentTime >= lyrics[i].time) {
      index = i;
    } else {
      break;  // 歌词已按时间排序，可以提前退出
    }
  }
  return index;
}
```

**同步机制**：

1. `audioRef` 的 `timeupdate` 事件触发时间更新
2. 调用 `findCurrentLineIndex()` 计算当前行索引
3. 行索引变化时触发自动滚动

### 歌词滚动

在 LyricsPage 中，歌词行变化时自动滚动到视口中央：

```typescript
useEffect(() => {
  const lineHeight = 52;
  const containerHeight = container.clientHeight;
  const scrollTo = currentLineIndex * lineHeight - containerHeight / 2 + lineHeight / 2;
  container.scrollTo({ top: Math.max(0, scrollTo), behavior: 'smooth' });
}, [currentLineIndex]);
```

### 歌词样式

| 状态 | 字号 | 字重 | 颜色 | 模糊 |
|------|------|------|------|------|
| 当前行 | 30px | semibold | 白色 | 无 |
| 已唱行 | 28px | normal | 白色 25% | 0.3px |
| 未唱行 | 28px | normal | 白色 40% | 0.3px |

## 进度条悬浮提示

在 [[页面与组件|AppleMusicDiscover]] 的 ProgressBar 组件中，鼠标悬停时显示详细提示框：

**提示框结构**：

```
┌─────────────────────────────────────┐
│ [封面]  歌曲标题 (13px bold)         │
│         艺人名 (11px light)          │
│         00:45 (12px bold)           │
│         / 03:18 (11px gray)         │
│         +0:45 (11px green/red)      │
└───────────▽─────────────────────────┘
```

- 最小宽度 260px
- 封面缩略图 40×40
- 底部居中三角指示器
- 150ms 淡入淡出过渡
- 时间差：正值绿色（快进），负值红色（回退）

## 添加新歌曲

向项目添加新歌曲需要同时更新两处：

1. **`mockData.ts` 的 `mockSongs` 数组**：添加完整的 Song 对象
2. **`AppleMusicDiscover.tsx` 的 `NEW_SONGS` 数组**：添加到新歌速递列表

本地歌曲还需在 `public/musics/index.json` 中添加条目，并将音频/封面文件放到 `public/musics/` 对应目录下。
